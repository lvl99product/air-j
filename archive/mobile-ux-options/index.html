UX/UI Component Analysis & Layout Options

Current Component Evaluation

1. Search & Day Review (Currently Action Buttons): Using full-width or half-width buttons for Search and Day Review takes up premium vertical space. Search is better suited as a standard top-bar input or icon. Day Review acts as a dashboard/summary, which fits perfectly as a Bottom Navigation item or a prominent floating trigger.

2. Category Filter Chips (Currently Horizontal Scroll): Chips/Pills are the correct component for filtering (excellent for discoverability). However, placing them directly above the Kanban columns can cause the screen to feel top-heavy.

3. To-Do / Doing / Done (Currently Segmented Control): Segmented Control is the correct native pattern for switching between 3 distinct list views on mobile. Swipeable tabs are exactly what users expect.

Conclusion: The individual components are mostly correct, but their grouping and placement causes cognitive overload and vertical crowding. Below are 4 structural options to resolve this.

Option 1: App Bar & Bottom Nav
Moves Day Review to a standard Bottom Nav. Search lives in the Top App Bar. Clean top-down hierarchy.
